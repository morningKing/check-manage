# 使用说明

## 1. 登录

访问系统后自动跳转到登录页面。

- **默认管理员账号：** admin / admin123
- 输入用户名和密码后点击「登录」或按 Enter 键
- 登录成功后跳转到首页

---

## 2. 首页

首页展示系统概览信息：

- **统计卡片：** 菜单数量、页面配置数、字段配置总数
- **快捷入口：** 菜单管理、页面配置的快速跳转链接
- **系统信息：** 技术栈和主要功能介绍

---

## 3. 数据页面

通过左侧菜单导航到各业务数据页面。每个数据页面根据页面配置自动生成表单和表格。

### 3.1 查看数据

- 页面加载后自动展示该集合的所有数据
- 表格列根据字段配置自动生成
- 序号列自动编号
- 支持关键字搜索（模糊匹配所有字段）
- reference 字段的继承字段会在表格中额外显示

### 3.2 新增数据

1. 点击页面右上角「+ 新增」按钮
2. 在弹出的对话框中填写表单
3. 必填字段会有红色星号标识
4. 如果页面绑定了校验脚本，提交时会自动执行校验（不通过则阻止提交并显示错误）
5. 点击「确定」保存，「取消」放弃

### 3.3 编辑数据

1. 在表格操作列点击「编辑」
2. 修改表单中的字段值
3. 提交时同样会执行绑定的校验脚本
4. 点击「确定」保存修改

### 3.4 删除数据

- **单条删除：** 在表格操作列点击「删除」，确认后执行
- **批量删除：** 通过表格左侧复选框选中多条记录，点击「批量删除」按钮，确认后删除所选记录
- 如果记录被其他数据引用（reference），会提示无法删除

### 3.5 导出 Excel

点击「导出」按钮，浏览器自动下载 .xlsx 文件。导出内容：
- 所有字段按配置顺序排列
- 下拉选项类字段显示为中文标签（而非值）
- 多选字段用「、」分隔

### 3.6 自定义脚本导出

如果页面绑定了导出脚本，「导出」按钮会变为下拉菜单：

- **导出 Excel**：默认 Excel 导出（同 3.5）
- **脚本名称**：执行管理员配置的自定义导出脚本，生成指定格式文件

### 3.7 行级导出

如果页面绑定了行级导出脚本，每行数据的操作列中会出现「导出」按钮：

- 单个行级脚本：直接显示「导出」按钮
- 多个行级脚本：显示下拉菜单，选择脚本执行
- 仅导出该行数据

### 3.8 导入 Excel

1. 点击「导入」下拉按钮
2. 选择「下载导入模板」获取 Excel 模板文件
3. 模板包含两个工作表：
   - **导入数据：** 按表头填写数据
   - **字段说明：** 字段名、类型、是否必填、可选值说明
4. 填写完成后，点击「导入」→「导入数据」选择文件
5. 系统解析文件并逐条导入，显示进度和结果
6. 导入时自动建立关联关系（关联字段通过显示值匹配目标记录）

## 4. 系统配置

> 仅管理员可访问。系统配置位于左侧菜单底部，分为三个子分组。

### 菜单结构

```
系统配置
├── 平台管理
│   ├── 菜单管理
│   ├── 页面配置
│   └── 用户管理
├── 数据工具
│   ├── 导出脚本
│   ├── 校验脚本
│   ├── ETL 管理
│   └── Open API
└── 系统运维
    ├── 操作日志
    └── 系统备份
```

所有系统配置菜单为内置保护菜单，不可编辑、删除或添加子菜单。

---

## 5. 菜单管理

> 系统配置 → 平台管理 → 菜单管理

管理侧边栏的菜单结构，支持最多 3 级嵌套。

### 5.1 页面布局

- **左侧：** 菜单树，展示当前菜单层级结构
- **右侧：** 选中菜单的编辑表单

### 5.2 系统菜单保护

「系统配置」及其下所有子菜单为内置系统菜单，受保护不可修改：

- 系统菜单节点右侧显示锁图标
- 无法添加子菜单、编辑或删除
- 点击系统菜单节点可查看其配置（只读模式）

### 5.3 新增菜单

- 点击「新增顶级菜单」创建根菜单
- 鼠标悬停在菜单节点上，点击「添加子菜单」创建子菜单
- 填写菜单信息后点击「创建」

### 5.4 编辑菜单

点击菜单树中的节点，右侧显示编辑表单：

| 字段 | 说明 |
|------|------|
| 菜单名称 | 侧边栏显示的名称（2-20 字符） |
| 菜单图标 | 从 Element Plus 图标库选择 |
| 路由路径 | 如 /inspection/case |
| 关联页面 | 叶子菜单需选择对应的页面配置 |
| 排序序号 | 控制同级菜单的显示顺序 |
| 可见角色 | 勾选 admin / developer / guest |

### 5.5 删除菜单

鼠标悬停在菜单节点上，点击「删除」，确认后删除。有子菜单时需先删除子菜单。

## 6. 页面配置

> 系统配置 → 平台管理 → 页面配置

定义数据页面的字段结构。每个页面配置对应一个数据集合。

### 6.1 页面布局

- **左侧：** 页面列表，支持搜索
- **右侧：** 页面基本信息 + 字段配置编辑器

### 6.2 创建页面

1. 点击左侧列表头部的「新增」按钮
2. 填写页面名称、描述、API 端点
3. 点击「创建」后自动选中新页面
4. 在字段编辑器中添加字段

### 6.3 字段配置

点击「添加字段」打开字段编辑对话框：

| 配置项 | 说明 |
|--------|------|
| 显示名称 | 表单标签和表格列标题 |
| 字段名 | 存储字段名（英文，如 caseName） |
| 控件类型 | 见下方控件类型列表 |
| 必填 | 表单提交时是否必须填写 |
| 主键 | 标记为唯一标识字段（支持复合主键） |
| 占位提示 | 输入框的提示文字 |

### 6.4 支持的控件类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| text | 单行文本 | 名称、编号 |
| textarea | 多行文本 | 描述、备注 |
| number | 数值 | 数量、金额 |
| select | 单选下拉 | 状态、类型 |
| multiSelect | 多选下拉 | 标签、多项选择 |
| radio | 单选按钮 | 二选一场景 |
| checkbox | 复选框 | 多项勾选 |
| date | 日期 | 日期字段 |
| datetime | 日期时间 | 精确到时分秒的时间字段 |
| file | 文件上传 | 文档附件（最多 5 个，单个 10MB） |
| image | 图片上传 | 照片（最多 9 张，单张 5MB） |
| relation | 多对多关联 | 与其他数据集合建立双向关联 |
| reference | 一对多引用 | 引用父记录，支持继承字段显示 |

### 6.5 选项配置（select / multiSelect / radio / checkbox）

支持两种选项来源：
- **静态选项：** 手动添加 标签-值 对
- **API 获取：** 配置 API 地址和标签/值字段名

### 6.6 关联字段配置（relation）

| 配置项 | 说明 |
|--------|------|
| 目标集合 | 要关联的数据集合 |
| 显示字段 | 下拉列表中显示的字段 |
| 反向字段 | 在目标集合中自动维护的反向关联字段名 |

关联是双向的：修改 A→B 的关联会自动同步 B→A。详见 [数据关联使用说明](数据关联使用说明.md)。

### 6.7 引用字段配置（reference）

| 配置项 | 说明 |
|--------|------|
| 目标集合 | 引用的父数据集合 |
| 显示字段 | 下拉列表中显示的字段 |
| 继承字段 | 在数据表格中额外显示的父记录字段 |

引用会建立依赖：被引用的父记录无法删除。引用字段支持点击跳转到被引用的数据页查看详情。

### 6.8 导出脚本绑定

在页面基本信息编辑区域可配置两类导出脚本：

| 配置项 | 说明 |
|--------|------|
| 导出脚本 | 选择页面级导出脚本（对全部数据导出），出现在页面「导出」下拉菜单中 |
| 行级导出 | 选择行级导出脚本（对单条记录导出），出现在每行操作列中 |

下拉列表根据脚本的导出维度自动过滤。

### 6.9 校验脚本绑定

在页面基本信息编辑区域可选择绑定校验脚本：

- 从已创建的校验脚本列表中选择
- 绑定后，该页面的数据在新增和编辑时会自动执行校验
- 校验不通过时阻止提交并显示错误信息

### 6.10 Open API 开关

在页面基本信息中可开启「允许 Open API 访问」：

- 开启后，外部系统可通过 API Key 访问该页面的数据
- 关闭时该集合对 Open API 不可见

## 8. 导出脚本管理

> 系统配置 → 数据工具 → 导出脚本

管理自定义导出脚本，将业务数据转换为指定格式的文件。

### 8.1 页面布局

- **左侧：** 脚本列表，显示名称、输出格式标签和导出维度标签
- **右侧：** 脚本编辑器（代码区 + 帮助文档标签页）

### 8.2 创建脚本

1. 点击左侧列表头部的「新增」按钮
2. 填写脚本信息：

| 字段 | 说明 |
|------|------|
| 脚本名称 | 显示在导出菜单中的名称 |
| 描述 | 脚本用途说明 |
| 输出格式 | json / csv / xml / txt / html |
| 导出维度 | 整页数据（对全部数据导出）或 单行数据（对单条记录导出） |
| 脚本代码 | Python 代码，处理 `data` 和 `fields` 变量，设置 `result` 输出 |

3. 切换输出格式时自动加载对应的脚手架代码模板
4. 点击「保存」创建脚本

### 8.3 脚本编写

脚本在服务端沙箱中执行，可使用以下预注入变量和模块：

**输入变量：**

| 变量 | 类型 | 说明 |
|------|------|------|
| `data` | list[dict] | 业务数据列表，每条为 `{字段名: 值}` |
| `fields` | list[dict] | 字段配置列表，含 label、fieldName、controlType、options 等 |
| `page_name` | str | 页面名称 |

**输出变量：**

| 变量 | 类型 | 说明 |
|------|------|------|
| `result` | str / bytes | **必须** — 导出文件内容 |
| `filename` | str | 可选 — 文件名（默认自动生成） |
| `content_type` | str | 可选 — MIME 类型（默认按格式推断） |

**可用模块：** json, csv, io, re, math, collections, xml.etree.ElementTree, xml.dom.minidom, datetime, timedelta

**安全限制：** 禁止 import、open()、exec()、eval()、`__` 属性访问。超时 10 秒。

### 8.4 测试脚本

保存脚本后，点击「测试」按钮可用模拟数据验证脚本执行：
- 成功：显示文件名、大小、MIME 类型和内容预览
- 失败：显示错误信息

### 8.5 删除脚本

点击「删除」并确认。删除脚本会自动解除所有页面配置中对该脚本的绑定。

## 9. 校验脚本管理

> 系统配置 → 数据工具 → 校验脚本

管理数据校验脚本，在数据新增和编辑时执行自定义校验逻辑。

### 9.1 页面布局

- **左侧：** 脚本列表，显示名称
- **右侧：** 脚本编辑器（CodeMirror Python 编辑器 + 帮助文档）

### 9.2 创建脚本

1. 点击左侧列表头部的「新增」按钮
2. 填写脚本名称和描述
3. 编写 Python 校验代码
4. 点击「保存」

### 9.3 脚本编写

校验脚本在沙箱中执行，可使用以下预注入变量：

**输入变量：**

| 变量 | 类型 | 说明 |
|------|------|------|
| `data` | dict | 当前提交的记录数据 |
| `fields` | list[dict] | 页面字段配置列表 |
| `all_data` | list[dict] | 该集合中的所有现有记录 |
| `is_edit` | bool | True 表示编辑操作，False 表示新增操作 |

**校验函数：**

| 函数 | 说明 |
|------|------|
| `add_error(field, message)` | 添加字段级错误，阻止提交 |
| `add_warning(field, message)` | 添加警告信息（不阻止提交） |

**关联数据函数：**

| 函数 | 说明 |
|------|------|
| `get_relation_data(field_name)` | 获取当前记录某关联字段的关联记录列表 |
| `get_all_relation_data(collection, field_name)` | 获取指定集合所有记录的关联数据 |

**示例：**

```python
# 检查名称唯一性
for existing in all_data:
    if existing.get('name') == data.get('name'):
        if is_edit and existing.get('id') == data.get('id'):
            continue
        add_error('name', '名称已存在，请使用其他名称')
        break

# 检查关联数据
relations = get_relation_data('templates')
if len(relations) == 0:
    add_warning('templates', '建议至少关联一个模板')
```

### 9.4 测试脚本

点击「测试」按钮，使用模拟数据验证脚本执行逻辑：
- 显示校验结果（错误和警告列表）
- 显示执行是否成功

### 9.5 绑定到页面

校验脚本创建后，需要在「页面配置」中绑定到具体页面才会生效（见 6.9 节）。

### 9.6 删除脚本

点击「删除」并确认。删除脚本会自动解除所有页面配置中对该脚本的绑定。

## 10. ETL 管理

> 系统配置 → 数据工具 → ETL 管理

ETL（Extract-Transform-Load）数据管道功能，用于从外部系统抽取数据，经转换后导入系统数据页。

### 10.1 页面布局

- **左侧：** ETL 任务列表，每项显示名称和状态标签（就绪/草稿）
- **右侧：** 任务编辑区（基本信息 + 管道编辑器 + 操作按钮 + 执行日志）

### 10.2 创建任务

1. 点击左侧列表头部的「新增」按钮
2. 填写任务名称和描述
3. 添加管道步骤
4. 点击「保存」

### 10.3 管道编辑器

管道以纵向步骤卡片流展示。每张卡片显示步骤类型图标、名称和配置摘要。

- **添加步骤：** 点击顶部「添加步骤」下拉按钮选择步骤类型
- **插入步骤：** 点击两个步骤之间的「+」按钮
- **编辑步骤：** 点击卡片上的编辑按钮，弹出配置对话框
- **删除步骤：** 点击卡片上的删除按钮
- **调整顺序：** 使用卡片上的上移/下移按钮

### 10.4 步骤类型

| 类型 | 图标 | 说明 |
|------|------|------|
| HTTP 请求 | Download | 调用外部 API 获取数据 |
| JSON 输入 | Document | 手动输入 JSON 数据（测试用） |
| Python 脚本 | Promotion | 自定义 Python 转换逻辑 |
| 字段映射 | Switch | 源字段名 → 目标字段名的批量重命名 |
| 条件过滤 | Filter | 通过 Python 表达式过滤记录 |
| 写入集合 | Upload | 将数据写入系统数据页 |

### 10.5 步骤配置详解

**HTTP 请求：**

| 配置项 | 说明 |
|--------|------|
| 请求 URL | 外部 API 地址 |
| 请求方法 | GET 或 POST |
| 请求头 | 动态键值对（如 Authorization） |
| 请求体 | POST 请求时的 Body 内容 |
| 响应路径 | 从 JSON 响应中提取数据的路径（如 `data.items`） |

**JSON 输入：**

直接输入 JSON 数组作为管道数据源，适合测试管道流程。

**Python 脚本：**

| 变量 | 说明 |
|------|------|
| `records` | 当前管道中的记录列表（list[dict]） |
| `result` | 脚本须设置此变量为转换后的记录列表 |

脚本可用模块同导出脚本（json, re, math, datetime 等），在沙箱中执行，超时 10 秒。

**字段映射：**

| 配置项 | 说明 |
|--------|------|
| 映射规则 | 源字段 → 目标字段的对应关系列表 |
| 保留未映射字段 | 关闭时只保留映射后的字段，开启时保留所有字段 |

**条件过滤：**

输入 Python 表达式，对每条记录求值，返回 True 则保留。表达式中可直接使用记录的字段名作为变量。

**写入集合：**

| 配置项 | 说明 |
|--------|------|
| 目标集合 | 从系统数据页中选择写入目标 |
| 写入模式 | insert（全部新增）/ upsert（有则更新无则新增）/ update（仅更新已有） |
| 匹配字段 | upsert/update 模式下用于匹配已有记录的字段 |

**错误处理：** 每个步骤可配置出错时的行为：
- **停止：** 立即终止整个管道
- **跳过：** 跳过当前步骤，继续执行后续步骤
- **继续：** 忽略错误，继续处理

### 10.6 测试运行

点击「测试运行」按钮执行管道但**不实际写入数据**：

- 弹窗显示各步骤执行状态（成功/失败/跳过）
- 每步显示处理的记录数变化
- 有错误时显示详细错误信息
- 用于验证管道配置是否正确

### 10.7 执行

点击「执行」按钮（需二次确认）正式运行管道：

- 数据实际写入目标集合
- 执行结果记录到执行日志
- 更新任务的最后运行时间和状态

### 10.8 执行日志

页面底部展示最近的执行记录：

- 执行时间、状态（成功/失败/部分成功）
- 处理记录数、成功数、错误数
- 可查看各步骤的详细执行结果

## 11. Open API

> 系统配置 → 数据工具 → Open API

管理外部系统通过 API 访问数据的密钥。

### 11.1 创建 API Key

1. 点击「创建 API Key」按钮
2. 输入密钥名称（如「外部系统A」）
3. 创建成功后显示完整密钥（**仅此一次可见**，请立即复制保存）

### 11.2 管理密钥

列表显示所有 API Key 的名称、创建时间、最后使用时间和状态。

操作：
- **启用/禁用：** 切换密钥状态
- **删除：** 永久删除密钥

### 11.3 外部系统接入

外部系统通过以下方式访问数据：

```
GET /api/v1/collections
  → 返回所有已开启 Open API 的数据集合列表

GET /api/v1/collections/{collection}/data
  → 返回指定集合的全部数据

请求头：
  X-API-Key: cm_xxxxxxxxxxxxxxxxxxxxxxxx
```

**前置条件：** 目标数据页需在「页面配置」中开启「允许 Open API 访问」。

## 13. 系统备份

> 系统配置 → 系统运维 → 系统备份

1. 点击「创建备份」按钮

4. 备份完成后出现在列表中，显示文件大小和记录数

### 13.2 下载备份

点击备份列表中的「下载」按钮，浏览器下载 .zip 文件。ZIP 文件内包含：
- `manifest.json` — 备份元信息
- 11 个 JSON 文件 — 各业务表数据（含导出脚本、校验脚本、API Key、ETL 任务、ETL 日志等）

### 13.3 从备份还原

1. 点击备份列表中的「还原」按钮

3. 确认后执行还原
4. 还原成功后页面自动刷新

> 还原是原子操作：要么全部成功，要么全部回滚，不会出现数据不一致。

### 13.4 上传还原

1. 点击「上传还原」按钮
2. 选择之前下载的 .zip 备份文件

4. 成功后页面自动刷新

### 13.6 删除备份

点击「删除」按钮并确认，备份文件和记录同时删除。

## 14. 常见操作流程

### 流程一：创建一个新的业务数据页面

1. **页面配置** → 点击「新增」→ 输入页面名称和 API 端点 → 创建
2. 在字段编辑器中逐个添加字段 → 配置字段类型和属性 → 保存配置
3. **菜单管理** → 选择父菜单 → 添加子菜单 → 填写名称、图标、路由路径 → 关联刚创建的页面 → 设置可见角色 → 创建
4. 刷新页面后侧边栏出现新菜单，点击即可进入数据页面

### 流程二：建立两个数据集合的关联

1. **页面配置** → 选择源页面 → 添加 relation 类型字段
2. 配置目标集合、显示字段、反向字段名 → 保存
3. 进入源数据页面 → 编辑记录 → 在关联字段下拉框中选择目标记录
4. 关联自动双向同步

### 流程三：建立父子引用关系

1. **页面配置** → 选择子页面 → 添加 reference 类型字段
2. 配置目标集合（父集合）、显示字段、继承字段 → 保存
3. 进入子数据页面 → 新增记录 → 在引用字段中选择父记录
4. 父记录的继承字段自动在表格中显示
5. 被引用的父记录受到删除保护

### 流程四：批量导入数据

1. 进入数据页面 → 点击「导入」→「下载导入模板」
2. 在 Excel 中按模板格式填写数据（参考「字段说明」工作表）
3. 点击「导入」→「导入数据」→ 选择文件
4. 查看导入结果（成功/失败条数）

### 流程五：配置自定义导出

1. **导出脚本管理** → 新增脚本 → 选择输出格式和导出维度 → 编写脚本代码 → 测试 → 保存
2. **页面配置** → 选择目标页面 → 在「导出脚本」或「行级导出」中绑定脚本 → 保存
3. 进入数据页面使用导出功能

### 流程六：配置数据校验

1. **校验脚本管理** → 新增脚本 → 编写校验逻辑 → 测试 → 保存
2. **页面配置** → 选择目标页面 → 在「校验脚本」中绑定脚本 → 保存
3. 该页面的数据新增和编辑时会自动执行校验

### 流程七：从外部系统导入数据（ETL）

1. **ETL 管理** → 新增任务 → 填写名称
2. 添加数据获取步骤（HTTP 请求或 JSON 输入）
3. 按需添加转换步骤（脚本转换、字段映射、条件过滤）
4. 添加写入步骤（写入集合，选择目标页面和写入模式）
5. 点击「测试运行」验证管道配置
6. 确认无误后点击「执行」正式导入

### 流程八：开放数据 API 供外部系统调用

1. **Open API** → 创建 API Key → 复制保存密钥
2. **页面配置** → 选择需开放的页面 → 开启「允许 Open API 访问」→ 保存
3. 将密钥和接口地址提供给外部系统

### 流程九：备份和迁移数据

1. **系统备份** → 创建手动备份 → 下载 ZIP 文件
2. 在目标环境部署系统 → 初始化数据库
3. **系统备份** → 上传还原 → 选择下载的 ZIP 文件
4. 确认还原 → 数据完整迁移

## 15. 注意事项

1. **密码安全：** 首次部署后请立即修改默认管理员密码
2. **备份策略：** 建议启用每日定时备份，保留至少 7 天
3. **删除操作：** 删除菜单前需先删除子菜单；删除被引用的数据会被阻止
4. **字段名规范：** 字段名只能使用英文字母、数字和下划线，以字母开头
5. **角色分配：** 遵循最小权限原则，非必要不授予 admin 角色
6. **浏览器支持：** 推荐使用 Chrome、Edge 等现代浏览器
7. **系统菜单：** 系统配置相关菜单为内置菜单，不允许修改或删除
8. **脚本安全：** 所有脚本（导出、校验、ETL）在沙箱中执行，禁止文件系统和网络访问（ETL HTTP 步骤除外），超时 10 秒自动终止
9. **API Key 安全：** API Key 创建后仅显示一次，请妥善保管；不用的密钥应及时禁用或删除
10. **ETL 执行：** 建议先使用「测试运行」验证管道配置，确认无误后再正式执行
