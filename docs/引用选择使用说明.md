# 引用选择（quoteSelect）使用说明

## 概述

**引用选择**是一种单向多选字段类型，允许一条记录引用另一个集合中的多条记录，表达"使用"关系。数据以 ID 数组形式直接存储在记录的 JSONB 字段中，无需独立的关联表。

### 适用场景

- 巡检记录引用多条巡检用例
- 任务关联多个参考文档
- 订单选择多个适用的优惠规则

## 与其他类型的对比

| 特性 | 关联关系 (relation) | 数据引用 (reference) | 引用选择 (quoteSelect) |
|------|---------------------|---------------------|------------------------|
| 方向 | 双向自动同步 | 单向（父→子） | 单向（仅源端保存） |
| 数量 | 多对多 | 一对多（单选） | 多选 |
| 继承字段 | 否 | 是 | 否 |
| 阻止删除 | 否 | 是 | 否 |
| 存储位置 | 独立 data_relations 表 | 记录 JSONB（单个 ID） | 记录 JSONB（ID 数组） |

## 配置步骤

### 1. 添加字段

1. 进入页面管理 → 选择目标页面 → 字段配置
2. 点击「添加字段」
3. 填写显示名称和字段名
4. 控件类型选择「引用选择」
5. 配置引用参数：
   - **目标集合**：选择要引用的数据集合
   - **显示字段**：选择在下拉列表中显示的字段（如"用例名称"）
6. 点击「确定」保存字段，然后点击「保存配置」

### 2. 使用方式

#### 新增/编辑记录

在表单中，引用选择字段以多选下拉框形式呈现：

- 支持搜索过滤
- 可选择多条目标记录
- 可清空选择

#### 表格展示

- 引用记录以 Tag 标签形式展示
- 最多显示 3 个标签，超出部分显示 "+N" 计数
- 点击标签可跳转到引用记录所在页面

#### 搜索

全局搜索可匹配引用记录的显示名称。

#### 导出/导入

- **导出**：引用选择列显示目标记录的主键值，用"、"分隔
- **导入**：在单元格中填写目标记录的主键值，用"、"分隔

## 注意事项

1. 引用选择是**单向**的——删除目标记录不会自动清理引用方的数据
2. 引用选择**不会阻止**目标记录被删除
3. 如果被引用的记录已删除，表格中会直接显示其 ID
4. 引用选择字段的数据直接存储在记录中，不需要额外的后端接口

## 顺序保证

引用选择字段**保证引用值的顺序在整个生命周期中保持不变**。

### 存储机制

引用值以有序 JSON 数组存储在 PostgreSQL JSONB 中：

```json
{"quotedCases": ["id-B", "id-C", "id-D"]}
```

JSON 数组是有序数据结构，数据库在写入和读取时都保持原始顺序。

### 各环节的顺序保证

| 环节 | 实现方式 | 顺序保证 |
|------|---------|---------|
| 表单选择 | El-Select 多选模式 v-model 绑定数组 | 新选项追加到末尾，取消选择不改变其余项的相对顺序 |
| 保存到后端 | PUT 请求体中的 JSON 数组 → 存入 JSONB | JSON 数组有序，JSONB 保持数组元素顺序 |
| 读取展示 | JSONB → Python list → JSON 响应 → `Array.map()` 解析标签 | 每一步都保持数组迭代顺序 |
| 表格 Tag | `v-for` 渲染 `_quote_*_labels` 数组 | 按数组顺序从左到右展示 |
| Excel 导出 | `Array.map().join('、')` | 按数组顺序拼接 |
| Excel 导入 | `String.split().map()` | 按分隔顺序生成数组 |

### 删除场景的顺序保证

当引用列表中间的某条记录被删除时，**其余记录的相对顺序保持不变**：

```
初始状态：A 引用 [B, C, D]（按此顺序保存）

场景1 — 编辑时取消选择 C：
  表单 v-model 从 [B, C, D] 变为 [B, D]
  保存后：A 引用 [B, D]（B 和 D 的相对顺序不变）

场景2 — 目标集合中 C 被删除：
  A 的存储数据仍为 [B, C, D]
  表格展示：B（名称）、id-C（ID fallback）、D（名称）
  下次编辑 A 并保存时：C 的 ID 不在选项列表中，自动清除为 [B, D]
```
