# 域⑥ 开放集成

> 架构图对应：业务域 ⑥（开放集成）。索引见 [`README.md`](./README.md)。

## 1. 模块职责

系统与外部世界的双向桥梁与事件驱动：**Open API**（外部系统按集合读写）、**Webhook**（数据事件触发出站通知）、**跨集合联动规则**（内部数据变更自动触发其它集合操作）。

## 2. 模块组成

| 蓝图/工具 | 职责 |
|-----------|------|
| `api_keys` | API Key 管理（生成/吊销） |
| `open_api` | 对外暴露集合数据 API（API Key 认证） |
| `webhook` | Webhook 规则 CRUD、测试、日志 |
| `utils/webhook_engine.py` | Webhook 执行（签名、重试、记录） |
| `trigger_rules` | 联动规则 CRUD、日志 |
| `utils/trigger_engine.py` | 联动触发执行 |

## 3. 数据模型

**`api_keys`** — Key 哈希、所属、状态。
**`webhook_rules`** — `source_collection`、`trigger_event`(create/update/delete/merge)、`trigger_condition` JSONB、目标 URL、`secret`、超时/重试。
**`webhook_logs`** — 执行历史（含重试）。
**`trigger_rules`** — `source_collection`、`trigger_event`、`trigger_condition`、`target_collection`、`action_type`(create/update/runScript)、`action_config` JSONB。
**`trigger_logs`** — 执行日志。

## 4. 核心流程

### 4.1 Open API
外部系统持 API Key（`X-Api-Key` / Bearer），经 `api_key_required` 校验后按集合读写 `dynamic_data`。详见 `docs/user-guide/integration/open-api.md`。

**文件字段访问**：`file`/`image` 字段上传的文件由 `data_files` 表 + 仓库外存储承载，记录里存 `{uid, url, ...}`。内部下载 `/api/data-files/<id>/download` 走 JWT，API Key 客户端访问不到。为此 Open API 增：
- `GET /api/v1/files/<id>` / `GET /api/v1/files/<id>/download`（`api_key_required`）——下载文件元数据/字节；
- 记录响应里给 `file`/`image` 字段的每个文件对象补 `apiUrl`（指向上面的下载端点），免去外部猜测内部 url；
- **安全边界**：仅当文件被**某个 `api_public` 集合**的记录引用时才可经 Open API 访问（防止用 Key 拉到挂在非公开集合上的文件）。

### 4.2 Webhook（出站事件）
[域③](./03-动态数据核心.md) 的 create/update/delete/merge 触发 `webhook_engine`：
1. 匹配 `source_collection` + 启用的规则，按 `trigger_condition`（如 `{"status":"completed"}`）过滤。
2. 向目标 URL POST payload，`X-Webhook-Signature` 携带 payload 的 **HMAC-SHA256**（用规则 secret 签名）。
3. 失败按配置超时/重试，结果写 `webhook_logs`。

### 4.3 跨集合联动（内部事件）
`trigger_engine` 在 CRUD 主操作后 `fire_triggers(event, collection, ...)`：
- 查匹配规则 → 检查 `trigger_condition` → 执行 `action`：
  - `create`：按 `fieldMapping` INSERT 目标集合；
  - `update`：按 `matchField/matchValue` UPDATE；
  - `runScript`：调 [域④](./04-脚本扩展.md) 沙箱。
- 字段映射表达式：`$source.<field>`（源记录值）、`$operator`（操作人）、`$NOW`（当前时间）、否则字面量。
- 每条规则结果写 `trigger_logs`。

## 5. 关键接口

| 路由 | 方法 | 鉴权 |
|------|------|------|
| `/apiKeys` ... | CRUD | admin.api_keys |
| `/open-api/...`（外部） | GET/POST | api_key_required |
| `/webhook` `/webhook/<id>` `/test` `/logs` | CRUD/POST/GET | admin.webhooks |
| `/triggerRules` `/<id>` `/logs` | CRUD/GET | login / admin.trigger_rules |

## 6. 依赖与协作关系

- **事件源**：[域③](./03-动态数据核心.md) 数据写操作是 Webhook 与联动的触发来源。
- **依赖**：[域④](./04-脚本扩展.md)（联动 runScript）、[域①](./01-认证与权限.md) 鉴权。
- **外部集成点**：Webhook 出站第三方端点；Open API 入站外部系统。

## 7. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| Webhook 完整性 | HMAC-SHA256 签名 | 接收端可校验来源与防篡改 |
| Webhook 可靠性 | 超时 + 重试 + 日志 | 网络不可靠，需可观测与补偿 |
| 联动规则 | 独立 `trigger_rules` 表 | 规则跨集合，不适合嵌入单个 PageConfig |
| 联动表达式 | `$source`/`$operator`/`$NOW` 占位 | 声明式映射，无需写代码 |
