# 域⑧ 运维监控

> 架构图对应：业务域 ⑧（运维监控）。索引见 [`README.md`](./README.md)。

## 1. 模块职责

平台的可观测与运维支撑：操作审计、系统备份与差异对比、用户通知、统计仪表盘。本域也是大多数**后台调度器**的归属说明处。

## 2. 模块组成

| 蓝图/工具 | 职责 |
|-----------|------|
| `operation_logs` | 操作审计日志（批次聚合、筛选、导出） |
| `backups` | 备份/还原、差异对比 |
| `utils/backup.py` | 备份/还原逻辑 |
| `notifications` | 用户通知 |
| `dashboards` | 仪表盘配置与聚合查询 |

前端：`OperationLog.vue`、`BackupManager.vue`、`BackupDiffDialog`、`NotificationBell`、`DashboardView`。

## 3. 数据模型

**`operation_logs`** — `action`、`target_type/id`、`description`、`field_changes` JSONB（字段级新旧值差异）、`created_at`。
**`backups`** / **`backup_settings`** — 备份记录与定时配置；备份文件落仓库外。
**`notifications`** — `user_id`、`type`(statusChange/mention/trigger/dependencyBroken…)、`title`、`content`、`source_collection`、`source_record_id`、`is_read`。
**`dashboards`** — `name`、`layout`(widget 数组)、`owner_id`、`is_global`。

## 4. 核心流程

### 4.1 操作审计与字段变更追踪
[域③](./03-动态数据核心.md) 的 PUT 处理器对比新旧 `data`，将有差异字段写入 `operation_logs.field_changes`（`[{field,label,from,to}]`），供时间线与审计页展示。

### 4.2 备份 / 还原 / 差异对比
- 手动/定时备份（`backup_scheduler`）**仅全量**：一律备份全库表（含 `dynamic_data.version` 等），已移除表级/选择性备份；可下载、跨环境迁移。还原为**整包全量**（`mode='replace'`，所有表一并替换），不支持挑表/部分还原。
- **差异对比**：当前数据 vs 历史备份、或两份备份之间，逐字段比较并可导出报告。还原后客户端需刷新以获取正确版本号。

### 4.3 通知
- **触发源**：状态变更（[域③](./03-动态数据核心.md)）、评论 @mention、联动（[域⑥](./06-开放集成.md)）、依赖状态变化（[域⑦](./07-版本协作.md)）。
- 通知创建包在 `try/except` 中，**永不阻塞主操作**。
- 前端 `NotificationBell` 30 秒轮询未读数，点击通知按 `source_collection/record_id` 精准跳转。

### 4.4 仪表盘聚合
`POST /dashboards/aggregate` 在 `dynamic_data` 上做 JSONB 聚合：`count`/`sum`/`avg`/`min`/`max`/分组统计，复用单表无需额外 ETL。

### 4.5 后台调度器（进程内常驻）
均在 `app.py` 启动，受 `WERKZEUG_RUN_MAIN` 守卫避免 Flask 重载双启：

| 调度器 | 周期 | 职责 | 归属域 |
|--------|------|------|--------|
| `backup_scheduler` | 配置 | 定时备份 | ⑧ 本域 |
| `dependency_scheduler` | 每小时 | 跨项目依赖复校验 | [⑦](./07-版本协作.md) |
| `ai_scan_scheduler` | 1 分钟 tick | 定时 AI 流水线调度 | [⑨](./09-AI智能助手.md) |
| `BatchWorker`（线程池3） | 事件驱动 | AI 批任务执行 | [⑨](./09-AI智能助手.md) |

## 5. 关键接口

| 路由 | 方法 | 鉴权 |
|------|------|------|
| `/operationLogs` | GET | admin.operation_logs |
| `/backups` `/<id>` `/restore` `/diff` | CRUD/POST | admin.backup |
| `/notifications` `/unread-count` `/<id>/read` `/read-all` | GET/PUT | login |
| `/dashboards` `/<id>` `/aggregate` | CRUD/POST | login / admin |

## 6. 依赖与协作关系

- **被写入**：接收 [域③](./03-动态数据核心.md) 审计、[域⑥](./06-开放集成.md)/[域⑦](./07-版本协作.md) 的通知来源。
- **依赖**：[域①](./01-认证与权限.md) 鉴权；备份覆盖全库（含各域表）。

## 7. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 字段变更 | 新旧对比写 `field_changes` | 审计可追溯到字段级 |
| 通知 | `try/except` 不阻塞主操作 | 通知是辅助，不能拖累核心 CRUD |
| 仪表盘聚合 | PostgreSQL JSONB 查询 | 复用单表，无需额外 ETL |
| 调度器 | 进程内 + WERKZEUG_RUN_MAIN 守卫 | 免外部依赖，避免重载双启 |
