# 域⑦ 版本协作

> 架构图对应：业务域 ⑦（版本协作）。索引见 [`README.md`](./README.md)。

## 1. 模块职责

项目级的分支隔离与跨项目协作：**项目版本管理**（分支/快照、合并、锁定）与**跨项目依赖**（声明项目间的数据依赖关系并校验、通知）。

## 2. 模块组成

| 蓝图/工具 | 职责 |
|-----------|------|
| `project_versions` | 项目分支/快照的创建、合并、删除、锁定 |
| `utils/project_version.py` | 合并逻辑、分支 CRUD |
| `cross_project_deps` | 依赖声明 CRUD、校验、合并依赖检查、删除保护 |
| `utils/cross_project_dependency.py` | 依赖校验逻辑 |
| `utils/dependency_scheduler.py` | 每小时周期依赖复校验 |
| `utils/notifier.py` | 依赖通知 |

前端：`components/common/ProjectVersionManager.vue`、`ProjectDependencyManager.vue`、`BeyondCompareMerge.vue`。

## 3. 数据模型

**`project_versions`** — 以 `projectMenuId` 关联项目；`versionType`(branch/snapshot)、`status`(active/merged/archived)、`parentVersion`、锁状态(`isLocked`/`lockedBy`/`lockedAt`)。创建分支时把该项目菜单下所有集合的记录复制一份并打上新 `branch_id`。
**`project_dependencies`** — 跨项目依赖声明，三 `relationType`：`track-main`/`read-write`/`read-only`。
**`project_dependency_relations`** — 每条依赖涉及的具体数据关联。

数据隔离落在 [域③](./03-动态数据核心.md) `dynamic_data.branch_id` 与 `data_relations.branch_id`。

## 4. 核心流程

### 4.1 分支生命周期
- **创建分支**：复制项目下全集合记录到新 `branch_id`。
- **锁定**：合并前可锁定分支防修改。
- **合并**：把分支记录复制回 `main`，按 `MergeStrategy`（`theirs`=分支胜 / `ours`=主干胜）处理冲突。
- **删除保护**：被跨项目依赖引用的分支不可删（`check_branch_delete_protection`）。
- **合并后**：重新校验依赖方依赖，状态变化则发通知。

数据页头部的「切换分支」下拉对**非访客**（开发/管理员）可见——后端切换分支接口为 `@write_required`，前端门禁已与之对齐；「管理版本」入口仍限管理员（`admin.project_versions`）。

### 4.2 跨项目依赖三模式

| 模式 | 行为 | 合并阻断 |
|------|------|----------|
| `track-main` | 自动跟随目标项目 main 分支 | 否 |
| `read-write` | 配套分支配对 | 目标分支未合并则阻断本方合并 |
| `read-only` | 钉住特定版本 | 否，但版本须存在 |

**校验触发**：依赖创建/更新、目标项目合并后、调度器周期校验。状态变化时向源项目管理员发通知（`dependencyBroken`/`dependencyWarning`/`dependencyResolved`），写 [域⑧](./08-运维监控.md) `notifications`。

## 5. 关键接口

| 路由 | 方法 | 鉴权 |
|------|------|------|
| `/project-versions` `/<id>` | CRUD | admin.project_versions |
| `/project-versions/<id>/switch` · `/<menu>/switch-main` | POST | **write_required** |
| `/project-versions/merge` `/<id>/lock` | POST | admin.project_versions |
| `/project-versions/<menu>/current-branch` `/<menu>` | GET | login |
| `/dependencies/...` | CRUD/校验 | login / admin.dependencies |

## 6. 依赖与协作关系

- **作用对象**：[域③](./03-动态数据核心.md) 的 `branch_id`（分支隔离）。
- **产出通知**：依赖状态变化经 `notifier` 写 [域⑧](./08-运维监控.md) 通知。
- **依赖**：[域①](./01-认证与权限.md) 鉴权；周期校验由进程内 `dependency_scheduler` 驱动（见 [域⑧](./08-运维监控.md) 调度器小节）。

## 7. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 隔离粒度 | 项目级分支（branch_id） | 跨多数据页一致切换，符合协作单元 |
| 分支创建 | 全量复制记录 | 简单可靠的隔离，避免写时复杂引用 |
| 切换分支权限 | write_required（非访客） | 开发需在分支上工作；管理版本才限管理员 |
| 依赖三模式 | track-main/rw/ro | 覆盖"自动跟随/配套协作/精确钉住"三类协作语义 |
| 删除保护 | 被依赖分支不可删 | 防止悬挂依赖 |
