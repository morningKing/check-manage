# 域④ 脚本扩展

> 架构图对应：业务域 ④（脚本扩展）。索引见 [`README.md`](./README.md)。

## 1. 模块职责

用户自定义 Python 脚本扩展点：**导出脚本**（把数据页/菜单数据导出为任意格式）与**校验脚本**（写库前自动校验）。二者均经 **script_runner 沙箱**安全执行（超时 + 禁危险内置）。

## 2. 模块组成

| 蓝图/工具 | 职责 |
|-----------|------|
| `export_scripts` | 导出脚本 CRUD、测试、执行、批量导出 |
| `validation_scripts` | 校验脚本 CRUD |
| `utils/script_runner.py` | 沙箱执行器（`run_export_script` / `run_menu_export_script` / `run_validation_script`） |
| 前端 `ExportScriptManager.vue` | 脚本编辑（CodeMirror）、scope 选择、测试面板、可复制脚本 ID |
| 工具 `scripts/debug_export_script.py` | 本地调试工具（拉真实数据 + 完整 traceback） |

## 3. 脚本维度（scope）

| scope | 注入变量 | 用途 |
|-------|----------|------|
| `page`（默认） | `data`、`fields`、`page_name` | 单数据页导出 |
| `row` | 同上（单行） | 行级导出 |
| `menu` | `menu_data`（多表 `{collection,pageName,records,fields,recordCount}`）、`menu_name`、`total_records` | 菜单级聚合导出，可产出多文件 |

校验脚本注入：`record`、`action`、`old_data`、`fields`、`collection`、`conn`，并提供 `add_error`/`add_warning`/`set_relations`/`query` 等回调。

## 4. 数据模型

**`export_scripts`** — `id`、`name`、`description`、`language`、`script`、`output_format`(json/csv/xml/txt/html)、`scope`。
**`validation_scripts`** — 脚本定义，绑定到页面，写操作时执行。

## 5. 核心流程

### 5.1 沙箱执行
`script_runner` 预注入安全 builtins 与模块（`json/csv/io/re/math/collections/ET/minidom/datetime/timedelta/pd/np`），并：
- `_validate_script` 拒绝 `import`、`from...import`、`open/exec/eval/...` 危险调用、双下划线属性。
- 行级脚本超时 60s，菜单/集合级 300s。
- 无 `print()`——脚本经 `result`（str/bytes）输出；菜单级 `result` 为 `[{filename, content}]` 列表。
- **单一命名空间执行**：`exec(script_code, namespace)` 只传一个 dict（同时作 globals/locals），注入变量与脚本顶层定义合并其中。**不可**写成 `exec(code, globals, locals)` 两个不同 dict——那样脚本以「类体作用域」运行，自定义函数 / 列表推导式 / lambda 等嵌套作用域只经 `LOAD_GLOBAL` 取名，看不到注入的 `data/fields/records/menu_data`，会报 `NameError`（即"不支持自定义函数/推导式"）。

### 5.2 scope 与代码一致性校验（保存时拦截）
菜单级脚本经 `run_menu_export_script` 执行，沙箱只注入 `menu_data`，**不注入** `fields/data/page_name`。若把页面级代码存成 `scope=menu`，运行期会抛 `NameError: name 'fields' is not defined`。故 `validate_export_script_scope` 在保存时拦截：`scope=menu` 必须引用 `menu_data`，否则返回 400 并提示改用菜单级写法或切回 page 维度。

### 5.3 校验脚本介入写流程
[域③](./03-动态数据核心.md) 的创建/更新在写库前调用绑定的校验脚本：返回 `errors` 则拒绝（400 + validationErrors），`warnings` 提示，`pending_relations` 在同事务内应用。

### 5.4 测试与调试
- **测试面板**：脚本管理页可选**真实数据页**执行（前 100 条），兼容 page/menu scope；不选则用硬编码样例。返回文件名/大小/记录数/预览。
- **本地调试工具** `scripts/debug_export_script.py`：拉真实数据在与沙箱一致的环境本地运行，输出完整 traceback；支持 `--script`(本地文件)/`--script-id`/`--script-name`、`--list-collections`、`--list-scripts`、`--limit`、`--out`。

## 6. 关键接口

| 路由 | 方法 | 鉴权 |

## 7. 依赖与协作关系

- **被调用**：[域③](./03-动态数据核心.md) 写前调校验脚本；[域⑤](./05-数据流转.md) 菜单导出与 [域②](./02-配置中心.md) 页面绑定的导出按钮调导出脚本；[域⑥](./06-开放集成.md) 联动 `runScript` 动作复用沙箱。
- **依赖**：[域①](./01-认证与权限.md) 鉴权；读 [域③](./03-动态数据核心.md) 数据与 [域②](./02-配置中心.md) 字段配置。

## 8. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 执行隔离 | 预注入白名单 + 禁危险语法 + 超时 | 用户脚本不可信，需安全边界 |
| 输出契约 | 赋值 `result`（无 print） | 沙箱无 IO，约定式返回 |
| scope 校验 | 保存时拦截 menu/page 错配 | 从源头杜绝运行期 NameError |
| 调试 | 真实数据测试面板 + 本地工具 | 样例数据测不出真实字段问题 |
