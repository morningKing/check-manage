# 域② 配置中心

> 架构图对应：业务域 ②（配置中心）。索引见 [`README.md`](./README.md)。

## 1. 模块职责

系统的**元数据层**，承载"配置驱动"理念：菜单树、页面字段 schema、列视图、系统配置、首页区块。[动态数据核心](./03-动态数据核心.md) 与前端动态路由都**读取**本域配置来决定 UI 渲染与 API 行为——新增业务实体只需写配置，无需建表或写页面。

## 2. 模块组成

| 蓝图 | 职责 | 配置载体 |
|------|------|----------|
| `menus` | 3 级菜单树（工作空间→项目→数据页），含 `roles` RBAC | `menus` 表 |
| `page_configs` | 页面字段 schema（驱动整个 UI/API） | `page_configs.fields` JSONB |
| `column_views` | 每页保存的列视图（可见性/顺序/宽度/筛选） | `column_views` 表 |
| `system_config` | 系统名称/Logo 等全局配置 | `system_config` 表 |
| `home_widgets` | 首页区块（欢迎/统计/快捷入口/Markdown/数据卡片/**快速录入表单**） | `home_widgets` 表 |

前端：`stores/menu.ts`（菜单树 shallowRef 缓存 + 角色过滤）、`stores/pageConfig.ts`（中央配置 store）、`stores/systemConfig.ts`、`views/admin/MenuManager.vue`、`PageConfigManager.vue`、`SystemSettings.vue`。

## 3. 数据模型

**`page_configs`** — `id`(=`page-<collection>`)、`name`、`fields` JSONB（字段数组）、`view_config`（看板等视图配置）、`export_scripts`/`row_export_scripts`（绑定的导出脚本）、`apiEndpoint`。

**`FieldConfig`**（`fields[]` 元素）核心属性：`fieldName`、`label`、`controlType`、`options`、以及可选的 `workflowConfig`（状态机）、`compositeTextConfig`、关联配置等。`controlType` 决定渲染组件与后端逻辑（见 [域③](./03-动态数据核心.md) 控件清单）。

**`menus`** — 树结构，`menuType` ∈ `workspace`(L1)/`project`(L2)/`data`(L3)；`data` 级有 `page_id` 指向 `page_configs`；`roles` JSONB 控菜单可见性。

**`home_widgets`** — `id`、`widget_type`、`title`、`content` JSONB、`enabled`、`order`、`visible_roles`。`content` 按类型异构：
- `data-card`：`{dataSource:{collection,branchId,filter,limit}, displayType:'count|list|table', columns, titleField, linkToDetail}`
- `quick-form`：`{targetCollection, buttonLabel, description, icon}` — 点击弹出目标数据页录入表单直接入库。

## 4. 核心流程

**配置 → UI/路由 自动生成**
1. 前端启动读 `menus` 表 → `dynamicRoutes.ts` 生成路由，统一指向 `DynamicPage.vue`。
2. 进入数据页 → 按 `pageId` 取 `page_configs.fields` → `DynamicForm`/`DataTable` 按 `controlType` 渲染。
3. 后端 `dynamic` 蓝图按同一 `collection` 提供 CRUD，校验主键唯一与字段。

**工作流配置（状态机）**：字段携带 `workflowConfig`（`transitions[]` 含 from/to/label/roles/conditions/actions），由 [域③](./03-动态数据核心.md) 写操作时校验状态转换合法性（角色门禁 + 前置条件 + 动作执行）。配置即代码，不新增表。

**看板配置**：`page_configs.view_config.kanban`（groupField/cardTitle/cardFields/columnOrder/cardColorField）驱动 `KanbanBoard` 分列渲染与拖拽。

**首页快速录入区块**：在 `SystemSettings → 首页配置` 配置 `targetCollection`，首页区块点击弹出该数据页 `DynamicForm`（自动过滤 autoSequence/autoTimestamp 字段），经 `pageConfigStore.addPageData` 入库（生成 id、填充自动字段、处理关联，与正常新增完全一致）。

## 5. 关键接口

| 路由 | 方法 | 鉴权 |
|------|------|------|
| `/menus` `/menus/<id>` | CRUD | login / admin.menus |
| `/pageConfigs` `/pageConfigs/<id>` | CRUD | login / admin.page_configs |
| `/column-views` | CRUD | login |
| `/system-config` | GET/PUT | login / admin.system_config |
| `/home-widgets` `/home-widgets/<id>` `/order` | CRUD | login / admin.home_widgets |

> 创建自定义区块仅允许 `custom-markdown` / `data-card` / `quick-form` 三类型。

## 6. 依赖与协作关系

- **被依赖**：[域③](./03-动态数据核心.md) 读字段 schema 决定渲染与校验；前端动态路由读 `menus`；[域①](./01-认证与权限.md) 读 `menus.roles` 判菜单可见性。
- **依赖**：[域①](./01-认证与权限.md) 鉴权；[域④](./04-脚本扩展.md)（页面绑定导出脚本）；首页快速录入复用 [域③](./03-动态数据核心.md) 的创建逻辑。

## 7. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 业务实体建模 | 配置（schema）而非建表/写页面 | 零迁移、零编码扩展新实体 |
| 工作流配置位置 | `FieldConfig.workflowConfig` | 跟随字段定义，配置即代码 |
| 看板配置位置 | `page_configs.view_config` | 页面级配置，不污染字段结构 |
| 列视图 | 独立表 `column_views` | 用户态视图与页面 schema 解耦 |
| 首页区块内容 | 异构 `content` JSONB | 一表容纳多种区块类型，新增类型零 schema 变更 |
