# 域⑤ 数据流转

> 架构图对应：业务域 ⑤（数据流转）。索引见 [`README.md`](./README.md)。

## 1. 模块职责

数据的进出与搬运：**ETL 数据管道**（从外部 HTTP 源抽取、转换、写入集合）、**菜单级批量导出**（多数据页打包 ZIP）、**文件/图片字段存储**（仓库外）。

## 2. 模块组成

| 蓝图/工具 | 职责 |
|-----------|------|
| `etl_tasks` | ETL 任务 CRUD、执行、日志 |
| `utils/etl_engine.py` | ETL 步骤编排执行引擎 |
| `menu_export` | 菜单级导出、批量清空、可导出菜单树/预览 |
| `data_files` | 数据页文件/图片字段的上传与下载 |

前端：`views/admin/data-migration/`（ExportTab/ImportTab/ClearTab）、ETL 管理页。

## 3. 数据模型

**`etl_tasks`** — 任务定义（步骤数组：抽取/转换/映射/过滤/写入）。
**`etl_logs`** — 执行历史。
**文件存储**（仓库外，`DATA_FILES_ROOT`）——数据页文件/图片字段实际 blob；`menu_export` 临时产物与 `backups` 见 [域⑧](./08-运维监控.md)。

## 4. 核心流程

### 4.1 ETL 管道
`etl_engine` 按任务定义的步骤顺序执行：
1. **抽取** — HTTP 从第三方源拉取（集成点）。
2. **转换** — 经 [域④](./04-脚本扩展.md) 沙箱脚本变换。
3. **字段映射 / 条件过滤**。
4. **写入集合** — upsert/update 模式写 `dynamic_data`；服务端批量写入跳过客户端版本校验但仍 `version+1`（见 [域③](./03-动态数据核心.md) 并发边界）。

### 4.2 菜单级导出
1. 选 N 个菜单 + 1 个导出脚本（覆盖菜单绑定）→ 预览各数据页记录数。
2. `execute_menu_export` 按脚本 `scope` 分流：`menu` 维度调 `run_menu_export_script`（一次拿全部表）；`page` 维度逐表调 `run_export_script`。
3. 各数据页产物按 `菜单名/文件名` 结构打包 ZIP 返回；文件名去重。

> 前端 `executeMenuExport` 经 axios 携带 JWT（早期曾因从错误 localStorage 键取 token 导致"未登录"，已修正为统一 `STORAGE_KEYS.AUTH_TOKEN`）。

### 4.3 数据页全量导出
数据页「导出 Excel」会先 `fetchPageData({loadAll:true})` 全量拉取（绕过分页 1000 条上限）再生成，保留当前筛选条件。

## 5. 关键接口

| 路由 | 方法 | 鉴权 |
|------|------|------|
| `/etlTasks` `/etlTasks/<id>` `/run` | CRUD/POST | admin.etl_tasks |
| `/menuExport` | POST | login（返回 ZIP） |
| `/menuExport/availableMenus` `/preview` `/batchClear` | GET/POST | login / admin |
| `/data-files/...` | 上传/下载 | login |

## 6. 依赖与协作关系

- **依赖**：[域④](./04-脚本扩展.md)（导出/转换脚本）、[域③](./03-动态数据核心.md)（读写数据）、[域②](./02-配置中心.md)（菜单/字段）、[域①](./01-认证与权限.md) 鉴权。
- **外部集成点**：ETL 抽取自第三方 HTTP 源。

## 7. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| ETL 写入 | 跳过客户端版本校验 + version+1 | 服务端批量，无并发竞争但保持一致 |
| 导出脚本复用 | 菜单导出复用脚本沙箱 | 一套脚本能力，page/menu 两维度 |
| 文件存储 | 仓库外 `DATA_FILES_ROOT` | 不入 git，且避开 AI 文件工具的 cwd 扫描范围 |
| 全量导出 | `loadAll=true` 绕分页上限 | 导出需完整数据而非当前页 |
