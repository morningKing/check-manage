# 版本管理交互改进设计

## 背景

当前版本管理使用右侧 Drawer（700px）弹出方式，存在以下问题：

1. **显示区域受限**：Drawer 宽度仅 700px，信息展示不完整
2. **无分页功能**：版本多时列表过长，滚动困难
3. **无搜索筛选**：查找特定版本效率低
4. **操作按钮密集**：5 个按钮并排，容易误触

## 设计目标

- 扩大展示区域，完整展示版本信息
- 添加分页功能，支持大数据量浏览
- 添加搜索和筛选，快速定位版本
- 优化操作按钮布局，减少误触

## 设计决策

### 交互形式

**选择：全屏弹窗模式（Dialog）**

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| 全屏弹窗 | 显示区域大、分页+搜索、仍为弹窗 | 遮挡当前页面 | ✅ 采用 |
| 独立页面 | 完整空间、可扩展、URL可分享 | 离开数据页面 | ❌ |
| 改进 Drawer | 保持上下文、改动小 | Drawer 局限性 | ❌ |

### 弹窗尺寸

**选择：80% 宽度 Dialog**

- 保留页面边缘可见，维持上下文感知
- 信息展示充足，适合大多数场景
- 高度自适应内容或固定 90%

### 操作按钮布局

**选择：常用按钮 + 下拉菜单**

```
操作列布局：
[切换] [更多 ▼]
         └── 对比
         └── 合并
         └── 恢复
         ──────────
         └── 删除（红色）
```

- **切换按钮**：高频操作，始终可见（仅分支类型显示）
- **更多菜单**：对比、合并、恢复、删除
- **已合并状态**：隐藏所有操作按钮，显示「已合并」标签

## 详细设计

### 弹窗结构

```
┌─────────────────────────────────────────────────────┐
│ 版本管理 - [Collection名称]                      ✕ │  ← 标题栏
├─────────────────────────────────────────────────────┤
│ ● 当前：主分支 / 分支名    [切换回主分支]           │  ← 当前分支状态
├─────────────────────────────────────────────────────┤
│ [搜索框] [状态筛选▼] [类型筛选▼] [搜索] [+创建版本] │  ← 工具栏
├─────────────────────────────────────────────────────┤
│ 版本名称 │ 类型 │ 状态 │ 数据 │ 创建者 │ 时间 │ 操作 │  ← 表头
│─────────────────────────────────────────────────────│
│ v1.0    │ 分支 │ 活跃 │ 45条 │ admin │ 01-15│[切换][更多]│
│ feature │ 分支 │ 活跃 │ 23条 │ dev1  │ 01-20│[切换][更多]│
│ v0.9    │ 快照 │ 已合并│ 50条 │ admin │ 01-10│   已合并   │
├─────────────────────────────────────────────────────┤
│ 共 28 个版本，当前显示 1-10    [每页:10▼] [上一页][1][2][3][下一页] │  ← 分页
└─────────────────────────────────────────────────────┘
```

### 功能规格

#### 搜索

- **搜索范围**：版本名称 + 描述（模糊匹配）
- **触发方式**：输入后点击「搜索」按钮或按 Enter

#### 筛选

- **状态筛选**：全部 / 活跃 / 已合并 / 已归档
- **类型筛选**：全部 / 快照 / 分支

#### 分页

- **默认每页**：10 条
- **可选**：10 / 20 / 50
- **显示**：总条数、当前范围、页码按钮

#### 表格列

| 列名 | 说明 | 宽度 |
|------|------|------|
| 版本名称 | 名称文本 + 类型标签 + 当前标签 | 自适应 |
| 类型 | 分支/快照标签 | 80px |
| 状态 | 活跃/已合并/已归档 | 80px |
| 数据 | 记录条数 | 80px |
| 创建者 | 用户名 | 100px |
| 创建时间 | YYYY-MM-DD HH:mm | 140px |
| 操作 | 切换按钮 + 更多下拉 | 140px |

#### 操作菜单项

根据版本状态动态显示：

| 状态 | 可用操作 |
|------|----------|
| 活跃分支 | 切换、对比、合并、恢复、删除 |
| 活跃快照 | 对比、恢复、删除 |
| 已合并 | 无（显示「已合并」标签） |
| 已归档 | 无 |

**菜单项样式**：
- 删除操作：红色文字，分隔线分隔
- 其他操作：默认颜色

### 创建版本

保留现有 Dialog 形式，在工具栏点击「+ 创建版本」弹出。

### 对比/合并/恢复

保留现有子弹窗形式（BackupDiffDialog、BeyondCompareMerge），从操作菜单触发。

## 组件修改

### VersionManager.vue

**修改点**：

1. 将 `el-drawer` 改为 `el-dialog`
   - `size="700px"` → `width="80%"`
   - `direction="rtl"` → 移除
   - 添加 `top="5vh"` 控制顶部距离

2. 添加搜索筛选工具栏
   - 搜索输入框
   - 状态/类型下拉选择器
   - 搜索按钮

3. 添加分页组件
   - `el-pagination`
   - 每页条数选择器

4. 修改操作列
   - 「切换」按钮始终显示（分支类型）
   - 其他操作改为 `el-dropdown` 下拉菜单

5. API 调用添加分页参数
   - `getVersions(collection, status, page, pageSize, keyword)`

### API 修改

#### getVersions 函数

添加分页和搜索参数：

```typescript
export function getVersions(
  collection?: string,
  status?: string,
  page?: number,
  pageSize?: number,
  keyword?: string
) {
  const params: Record<string, string | number> = {}
  if (collection) params.collection = collection
  if (status) params.status = status
  if (page) params.page = page
  if (pageSize) params.pageSize = pageSize
  if (keyword) params.keyword = keyword
  return get<CollectionVersion[]>('/versions', params)
}
```

#### 后端路由修改

`server/routes/versions.py` 的 `get_versions_route` 添加分页支持：

- 接收 `page`、`pageSize`、`keyword` 参数
- 返回格式改为 `{ items: [...], total: number }`
