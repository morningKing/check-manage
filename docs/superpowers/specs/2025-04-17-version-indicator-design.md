# 数据页面版本标识显示设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在数据页面和编辑/查看对话框中显示当前分支信息，让用户清楚数据来源。

**Architecture:** 在DynamicPage.vue组件中添加版本状态获取和显示逻辑，通过API获取当前用户分支信息，在页面标题旁显示分支标签，提供快捷切换入口。编辑/查看对话框标题右侧同步显示分支标签。

**Tech Stack:** Vue 3 + TypeScript + Element Plus (前端), 已有的版本API (getCurrentBranch)

---

## 设计决策

### 1. 数据主页版本标识

**选择：标题旁标签**

在页面标题 `<h2>{{ pageConfig?.name }}</h2>` 旁边显示当前分支标签：
- 主分支：绿色标签「主分支」
- 其他分支：蓝色标签显示分支名称

**理由：**
- 紧凑简洁，不占用额外空间
- 与标题自然关联，易于理解
- 标签样式醒目但不干扰

### 2. 分支切换交互

**选择：标签旁下拉按钮**

在分支标签旁边显示「切换 ▼」按钮：
- 点击弹出下拉菜单，列出所有可选分支
- 菜单底部有「管理版本...」入口跳转版本管理弹窗

**理由：**
- 明确的切换入口，操作意图清晰
- 下拉菜单提供快速切换，无需打开版本管理弹窗
- 保留版本管理入口满足复杂操作需求

### 3. 编辑/查看对话框版本

**选择：对话框标题旁显示**

在编辑/查看对话框标题右侧显示当前分支标签：
- 格式与主页一致（绿色/蓝色标签）
- 用户编辑数据时清楚当前所在分支

**理由：**
- 与主页风格统一
- 不增加对话框高度
- 标题旁位置醒目但不干扰内容区域

## 数据流

1. **页面加载时**
   - DynamicPage.vue 调用 `getCurrentBranch(collection)` API
   - API 返回 `{ branchId, branchName }`
   - 如果 `branchId` 为 null，表示主分支

2. **分支标签渲染**
   - 主分支：显示「主分支」绿色标签
   - 其他分支：显示分支名称蓝色标签

3. **切换下拉菜单**
   - 调用 `getVersions(collection)` 获取所有分支列表
   - 过滤 `versionType === 'branch'` 的版本
   - 渲染下拉菜单选项

4. **切换操作**
   - 用户选择分支 → 调用 `switchToVersion(branchId)`
   - 成功后刷新当前页面数据
   - 更新分支标签显示

## API 使用

已有API，无需新增：

| API | 用途 |
|-----|------|
| `getCurrentBranch(collection)` | 获取当前用户分支 |
| `getVersions(collection)` | 获取版本列表（含分支） |
| `switchToVersion(versionId)` | 切换到指定分支 |
| `switchToMainBranch(collection)` | 切换回主分支 |
