# 版本合并交互重设计

## 概述

重新设计版本合并功能的用户交互，解决当前三步向导模式的体验问题：
- 步骤繁琐：即使简单合并也需走完三步
- 信息展示不直观：需要来回切换才能看到完整状态
- 字段选择体验差：修改记录的字段级选择过于复杂

## 设计方案：单页统一视图

核心思路：**去掉向导，一切在一页完成**

### 整体布局

├─────────────────────────────────────────────────────────────────┤

│                                                                  │
│  Main Content Area (可滚动)                                       │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 新增记录 Section                                           │  │
│  └───────────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 删除记录 Section                                           │  │
│  └───────────────────────────────────────────────────────────┘  │

**布局参数：**
- 对话框宽度：1000px
- 统计条固定在顶部，滚动时始终可见
- 三个 Section 默认全部展开，可手动折叠

**交互语义：**

| 记录类型 | 选中 | 未选中 | 语义 |
|---------|------|--------|------|
| 新增 | ☑ | ☐ | 选中=接受新增，未选中=忽略（不添加） |
| 删除 | ☑ | ☐ | 选中=确认删除，未选中=保留 |

**设计细节：**
- 每条记录一行，左侧复选框，右侧显示主字段值
- 标题栏的「全选」按钮为双态切换：全选 ↔ 全不选
- 选中的行背景色高亮（新增绿色调，删除红色调）

### 修改记录交互

```

│     ┌─────────────────────────────────────────────────────┐   │

│                                                                │
│  ☑ 记录C-003    巡检任务-南区-2024003                          │
│     └─ 字段选择已折叠（使用源版本: 2字段, 目标版本: 1字段）      │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**两级选择机制：**

1. **记录级选择**
   - 选中记录 → 该记录参与合并
   - 取消选中 → 该记录不参与合并

2. **字段级选择**（默认折叠，用户点击展开）
   - 三列：源版本值、目标版本值、单选按钮
   - 默认选中"源版本"（保守策略：接受变更）
   - 特殊字段（如更新时间）自动标记，不可选

**快捷操作：**
- 每条记录可单独"全部使用源/目标"
- Section 级别的"全选"：选中所有修改记录，字段默认使用源版本

### 批量操作逻辑

**顶部批量按钮：**

| 按钮 | 点击后效果 |
|------|-----------|
| 全部接受源版本 | 新增：全选<br>删除：全选<br>修改：全选，所有字段选"源" |
| 全部接受目标版本 | 新增：全不选<br>删除：全不选<br>修改：全选，所有字段选"目标" |

**Section 级别的"全选"按钮：**
- 双态切换：全选 ↔ 全不选
- 修改记录全选时，字段默认使用源版本

### 底部确认区

```
├─────────────────────────────────────────────────────────────────┤
│  💡 已选择 12 项变更：新增 5 / 删除 2 / 修改 5                    │
│                                                  [取消] [确认合并] │
└─────────────────────────────────────────────────────────────────┘
```

**状态变化：**

| 选择状态 | 底部提示 | 确认按钮 |
|---------|---------|---------|
| 未选择任何变更 | "⚠️ 请至少选择一项变更" | 禁用（灰色） |
| 已选择变更 | "💡 已选择 X 项变更：新增 A / 删除 B / 修改 C" | 可点击 |
| 提交中 | "正在合并..." | 加载状态 |

**确认后处理：**
1. 点击「确认合并」→ 按钮 loading
2. 成功 → Toast 提示"合并成功"，对话框关闭，刷新列表
3. 失败 → Toast 提示错误信息，保持对话框

## 术语定义

### 版本方向

在合并场景中：
- **源版本（Source）**：被合并的版本，即用户点击"合并"按钮的那个版本
- **目标版本（Target）**：当前工作区，即合并结果存放的位置

### 字段值映射

`DiffChangedField` 类型中的字段：
- `oldValue`：目标版本（当前工作区）的字段值
- `newValue`：源版本（被合并版本）的字段值

**示例**：将版本A合并到当前工作区
```
DiffChangedField {
  fieldName: 'status',
  oldValue: '进行中',    // 当前工作区的值
  newValue: '已完成'     // 版本A的值
}
```

选择"源版本"时使用 `newValue`，选择"目标版本"时使用 `oldValue`。

## 特殊字段处理

### 自动字段的识别规则

以下字段类型在合并时自动处理，不参与用户选择：

| 字段特征 | 自动处理规则 | 示例 |
|---------|-------------|------|
| `controlType === 'autoSequence'` | 保持当前工作区的值，不合并 | 编号字段 |
| `fieldName === 'createTime'` | 保持原创建时间 | 创建时间 |
| `fieldName === 'createUser'` | 保持原创建人 | 创建人 |
| `fieldName === 'updateTime'` | 使用合并时的当前时间 | 更新时间 |
| `fieldName === 'updateUser'` | 使用当前操作用户 | 更新人 |

**UI 显示**：
- 自动字段在字段选择表中显示为灰色
- 选择列显示 "-" 或 "自动"
- 不可点击选择

## 实现要点

1. **性能优化**：
   - 记录数 > 50 时启用虚拟滚动
   - 使用 `el-virtual-scroll` 或自定义实现

2. **字段显示**：根据 FieldConfig 获取字段标签和格式化方式

3. **错误处理**：合并失败时保留用户选择状态，支持重试

4. **关闭保护**：有选择时关闭对话框需确认
