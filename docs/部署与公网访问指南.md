# 部署与公网访问指南

本文档说明如何在本地部署 check-manage 项目，并通过 frp 内网穿透对外暴露服务。

---

## 一、本地部署

### 1.1 环境要求

| 依赖 | 版本要求 |
|------|----------|
| Node.js | >= 18 |
| Python | >= 3.9 |
| PostgreSQL | >= 14 |

### 1.2 安装依赖

```bash
# 前端依赖
npm install

# 后端依赖
pip install flask flask-cors psycopg2-binary PyJWT openpyxl werkzeug
```

### 1.3 初始化数据库

编辑 `server/config.py`，确认数据库连接信息：

```python
DB_CONFIG = {
    'host': 'localhost',
    'dbname': 'casemanage',
    'user': 'postgres',
    'password': 'your-password',
    'port': 5432,
}
```

执行初始化：

```bash
cd server && python init_db.py
```

### 1.4 启动服务

```bash
# 一键构建 + 启动（推荐）
./start.sh

# 跳过前端构建（已有 dist/ 时）
./start.sh --skip-build
```

启动后本地访问地址：`http://localhost:8080`

默认管理员账号：`admin` / `admin123`

### 1.5 服务架构

```
浏览器 → localhost:8080 (反向代理)
            ├── 静态文件 ← dist/
            └── /api/*   → Flask 后端 (localhost:3001)
                              └── PostgreSQL (localhost:5432)
```

## 二、frp 内网穿透配置

通过 frp 将本地 8080 端口暴露到公网，实现外部访问。

### 2.1 准备工作

1. 一台有公网 IP 的服务器（如阿里云、腾讯云）
2. 下载 frp：https://github.com/fatedier/frp/releases（服务器和本地各一份）
3. （可选）一个域名，A 记录解析到服务器 IP

### 2.2 服务端部署（公网服务器）

将 `deploy/frps.toml` 上传到服务器，修改配置：

```toml
# frps.toml

bindPort = 7000                              # frp 通信端口
vhostHTTPPort = 80                           # HTTP 代理端口
auth.token = "your-secret-token"             # 改为自定义密钥

# 管理面板（可选）
webServer.addr = "0.0.0.0"
webServer.port = 7500
webServer.user = "admin"
webServer.password = "your-dashboard-password"
```

启动：

```bash
./frps -c frps.toml
```

需要开放的服务器防火墙端口：

| 端口 | 用途 |
|------|------|
| 7000 | frp 客户端连接 |
| 80 | HTTP 代理入口（浏览器访问） |
| 7500 | 管理面板（可选） |

### 2.3 客户端部署（本地机器）

修改 `deploy/frpc.toml`：

```toml
# frpc.toml

serverAddr = "123.45.67.89"                  # 改为服务器公网 IP
serverPort = 7000
auth.token = "your-secret-token"             # 与服务端一致

[[proxies]]
name = "check-manage-http"
type = "http"
localIP = "127.0.0.1"
localPort = 8080
customDomains = ["check.your-domain.com"]    # 改为你的域名
```

启动顺序：

```bash
# 1. 先启动项目
./start.sh

# 2. 再启动 frpc
./frpc -c frpc.toml
```

### 2.4 访问方式

**方式一：HTTP + 域名（推荐）**

将域名 A 记录解析到服务器 IP，然后通过域名访问：

```
http://check.your-domain.com
```

**方式二：TCP + 端口（无需域名）**

修改 `frpc.toml`，注释掉 HTTP 代理，启用 TCP 代理：

```toml
# 注释掉 HTTP 代理
# [[proxies]]
# name = "check-manage-http"
# type = "http"
# ...

# 启用 TCP 代理
[[proxies]]
name = "check-manage-tcp"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8080
remotePort = 18080
```

服务器防火墙额外开放 18080 端口，通过 IP 访问：

```
http://123.45.67.89:18080
```

## 三、设为系统服务（可选）

### 3.1 Linux 服务端 — systemd

创建 `/etc/systemd/system/frps.service`：

```ini
[Unit]
Description=frps service
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/frps -c /etc/frp/frps.toml
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable frps
sudo systemctl start frps
```

### 3.2 Windows 客户端 — 开机自启

创建 `start-frpc.bat`：

```bat
@echo off
cd /d %~dp0
frpc.exe -c frpc.toml
```

按 `Win + R` → 输入 `shell:startup` → 将 bat 文件的快捷方式放入启动文件夹。

## 五、注意事项

1. **安全性**：`auth.token` 务必修改为高强度密钥，避免使用默认值
2. **端口冲突**：服务器 80 端口如被 Nginx 等占用，可将 `vhostHTTPPort` 改为其他端口
3. **防火墙**：确保服务器安全组 / iptables 放行所需端口
4. **稳定性**：生产环境建议将 frps 和 frpc 都配置为系统服务，实现断线自动重启
5. **HTTPS**：如需 HTTPS，建议在服务器用 Nginx 反向代理 frps 的 HTTP 端口，由 Nginx 处理 SSL 证书
