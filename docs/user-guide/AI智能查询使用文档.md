# AI 智能查询使用文档

## 概述

AI 智能查询功能允许用户通过**自然语言**描述数据筛选需求，系统自动将其转换为查询条件并执行。无需手写 JSON 查询语法，只需像跟人说话一样描述你要找什么数据。

**底层原理**：系统将页面的字段配置（名称、类型、选项值等）发送给阿里百炼 LLM，由 AI 将自然语言翻译为 MongoDB 风格的 filter JSON，再通过已有的查询引擎执行。

---

## 快速开始

### 1. 进入 AI 搜索模式

在任意数据页面，点击搜索栏右侧的 **魔法棒图标按钮**（黄色高亮表示已激活）。

三种搜索模式互斥切换：
- **简单搜索**（默认）— 关键字模糊匹配
- **高级查询** — 手写 MongoDB JSON 语法
- **AI 智能查询** — 自然语言描述

### 2. 输入查询描述

在文本框中用自然语言描述你的筛选需求，然后按 **Enter** 或点击 **「AI 查询」** 按钮。

### 3. 查看结果

- 查询成功后，数据表格自动更新为筛选结果
- 搜索栏出现橙色 **「AI 筛选条件」** 标签，鼠标悬停可查看生成的 filter JSON
- 记录计数显示 **（已筛选）** 标记

### 4. 清除筛选

点击 **「清除」** 按钮或关闭 **「AI 筛选条件」** 标签即可恢复全量数据。

## 查询示例

以下示例以巡检用例页面为例（字段包括：用例名称、状态、优先级、创建人等）。

| 自然语言输入 | AI 生成的 filter |
|---|---|
| 找出所有待评审的用例 | `{"status": "待评审"}` |
| 优先级为高的用例 | `{"priority": "high"}` |
| 创建人是张三的记录 | `{"creator": {"$regex": "张三"}}` |
| 状态不是已完成的用例 | `{"status": {"$ne": "已完成"}}` |
| 优先级为高或中的用例 | `{"priority": {"$in": ["high", "medium"]}}` |
| 名称包含"登录"的用例 | `{"name": {"$regex": "登录"}}` |
| 已完成且优先级高的用例 | `{"status": "已完成", "priority": "high"}` |
| 找出没有填写负责人的记录 | `{"assignee": {"$exists": false}}` |

> **提示**：AI 会自动将中文字段名映射为系统内部字段名，将选项的中文标签转换为对应的 value 值。

## 使用技巧

### 描述越具体，结果越准确

| 效果 | 示例 |
|---|---|
| 较好 | "找出状态为待评审且优先级为高的用例" |
| 一般 | "找一些重要的待审用例" |

### 支持的查询能力

- **精确匹配** — "状态是已完成"
- **模糊搜索** — "名称包含登录"
- **比较运算** — "分数大于80"、"创建时间晚于2024-01-01"
- **列表匹配** — "状态是待评审或评审中"
- **取反条件** — "状态不是已废弃"
- **存在性判断** — "没有填写负责人的记录"
- **组合条件** — "高优先级的且状态为待评审的用例"

### 不支持的场景

以下字段类型不参与 AI 查询（AI 不会看到这些字段的 schema）：

- `relation` — 关联字段（多对多）
- `reference` — 引用字段（父子引用）
- `quoteSelect` — 引用选择
- `file` / `image` — 文件/图片
- `richText` — 富文本

## 常见问题

### Q: AI 返回"未能生成有效的查询条件"？

描述太模糊或与页面字段无关。请参考页面上的实际字段名称，用更具体的描述重试。

### Q: 查询结果不符合预期？

鼠标悬停 **「AI 筛选条件」** 标签查看生成的 filter JSON，确认 AI 的理解是否正确。如果不对，可以：
1. 调整自然语言描述，使其更明确
2. 切换到高级查询模式，手动修改 filter JSON

### Q: 提示"AI 服务未配置"？

管理员需要在 `server/config.py` 的 `AI_CONFIG` 中配置有效的 DashScope API Key。

### Q: 响应较慢？

AI 查询需要调用外部 LLM 服务，通常需要 2-5 秒。网络状况和 LLM 服务负载会影响响应时间。

## API 接口

供前端或第三方集成调用。

### POST /api/ai/query

**功能**：将自然语言翻译为 MongoDB 风格的 filter JSON（只翻译，不执行查询）。

**请求头**：`Authorization: Bearer <token>`

**请求体**：

```json
{
  "collection": "inspection-case",
  "question": "找出所有待评审的用例"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `collection` | string | 是 | 集合标识（pageId 去掉 `page-` 前缀） |
| `question` | string | 是 | 自然语言查询描述 |

**成功响应** (200)：

```json
{
  "filter": {
    "status": "待评审"
  }
}
```

**错误响应**：

| 状态码 | 场景 | 示例 |
|--------|------|------|
| 400 | 参数缺失 | `{"error": "collection 和 question 不能为空"}` |
| 404 | 页面配置不存在 | `{"error": "未找到页面配置: xxx"}` |
| 422 | AI 生成的语法无效 | `{"error": "AI 生成的查询语法无效: ..."}` |
| 503 | API Key 未配置 | `{"error": "AI 服务未配置 API Key"}` |

**调用示例**：

```bash
curl -X POST http://localhost:3001/ai/query \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{"collection": "inspection-case", "question": "找出所有高优先级的用例"}'
```
