# 通知中心

系统通过通知中心向用户推送重要事件。入口为顶部导航栏的**铃铛图标**，未读数量以角标显示。

## 通知来源

| 类型 | 触发场景 |
|------|----------|
| 工作流待办 | 跨页工作流推进到某阶段、需要你（按角色）处理时 |
| 依赖告警 `dependencyBroken` | 跨项目依赖被破坏（如目标分支未合并即合并了源项目） |
| 依赖警告 `dependencyWarning` | 依赖处于风险状态（如目标版本即将失效） |
| 依赖恢复 `dependencyResolved` | 此前异常的依赖恢复正常 |
| 其它系统通知 | 备份、合并校验等系统事件 |

## 查看与处理

1. 点击顶部 **铃铛图标** 打开通知列表。
2. 列表按时间倒序展示，未读项高亮。
3. 点击某条通知可跳转到相关页面（如对应的待办记录、依赖详情）。
4. 可将单条标记为已读，或一键 **全部已读**。

## 工作流待办

工作流相关通知也可在 **工作流收件箱**（`WorkflowInbox`）集中处理，按角色过滤出「轮到你处理」的记录。详见 [工作流引擎指南](../data/workflow.md)。

## 依赖通知

跨项目依赖的状态变化会通知**源项目的管理员**。配置与排查见 [跨项目依赖说明](../integration/cross-project-deps.md)。
