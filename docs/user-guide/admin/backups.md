# 系统备份与还原

系统支持对整个数据库做**全量备份**，并可在需要时**还原**或对比两次备份的差异。备份功能位于 **设置中心 → 系统运维（`/admin/sys-ops`）→ 备份**，需要管理员权限（`admin.backups`）。

## 备份类型

- **手动备份**：随时点击「创建备份」即时生成一份全量快照。
- **定时备份**：在「备份设置」中配置周期（如每日某时刻），由后台调度器（`backup_scheduler`）自动执行。

> 备份为**全量**：覆盖所有业务数据、配置、关系等。备份文件存放在仓库外的 `backups` 目录，不进入版本库。

## 创建备份

1. 进入 **系统运维 → 备份**。
2. 点击 **创建备份**，可填写备注说明。
3. 等待生成完成，新备份出现在列表，显示创建时间、大小、备注。

## 还原

1. 在备份列表选择目标备份，点击 **还原**。
2. 确认对话框中再次确认（还原会用备份内容覆盖当前数据库，**操作不可逆**，建议还原前先创建一份当前状态的备份）。
3. 还原完成后刷新页面。

## 差异对比

选择两份备份可查看其差异，用于在还原前确认改动范围（新增 / 删除 / 修改的记录与配置）。

## 定时备份设置

在「备份设置」中：

- **启用定时备份**：开关。
- **执行周期**：按计划触发的时间。
- **保留份数**（如有）：超出后自动清理最旧的备份。

## 常见问题

- **还原后数据没变化**：确认选择的是正确的备份；还原是整库覆盖，若备份与当前一致则无可见变化。
- **备份失败**：检查磁盘空间与 `backups` 目录可写权限。
