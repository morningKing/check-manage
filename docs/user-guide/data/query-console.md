# 数据查询控制台使用文档

## 概述

数据查询控制台（路径：系统配置 → 数据工具 → 数据查询）提供 MongoDB 风格的查询语法，支持：

- 对任意集合执行条件查询
- 跨集合连表查询（$lookup）
- 字段投影（select）
- 排序（sort）
- 分页与导出

---

## 查询语句结构

```json
{
  "collection": "集合标识",
  "query": { 查询条件 },
  "lookup": [ 连表配置 ],
  "select": [ "字段1", "字段2" ],
  "sort": { "字段": 1 },
  "limit": 200
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `collection` | string | 是 | 目标集合标识（如 `inspection-case`） |
| `query` | object | 否 | MongoDB 风格查询条件，默认 `{}` 查全部 |
| `lookup` | array | 否 | 连表查询配置，可配多个 |
| `select` | array | 否 | 返回的字段列表（投影），不填则返回全部字段 |
| `sort` | object | 否 | 排序，`1` 升序、`-1` 降序 |
| `skip` | number | 否 | 跳过记录数（分页用，默认 `0`） |
| `limit` | number | 否 | 返回上限（默认 `200`，最大 `2000`） |

> 所有字段名均支持**中文标签**和**英文字段名**两种写法，系统自动映射。

---

## 查询操作符

### 比较

| 操作符 | 说明 | 示例 |
|--------|------|------|
| （隐式） | 精确等于 | `{"用例ID": "IC-001"}` |
| `$eq` | 等于 | `{"用例ID": {"$eq": "IC-001"}}` |
| `$ne` | 不等于 | `{"状态": {"$ne": "已删除"}}` |
| `$gt` | 大于 | `{"年龄": {"$gt": 18}}` |
| `$gte` | 大于等于 | `{"分数": {"$gte": 60}}` |
| `$lt` | 小于 | `{"价格": {"$lt": 100}}` |
| `$lte` | 小于等于 | `{"价格": {"$lte": 99.9}}` |

### 范围组合

多个操作符可在同一字段上组合使用：

```json
{ "分数": { "$gte": 60, "$lt": 90 } }
```

### 数组

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `$in` | 值在列表中 | `{"状态": {"$in": ["进行中", "已完成"]}}` |
| `$nin` | 值不在列表中 | `{"状态": {"$nin": ["已删除"]}}` |

### 字符串

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `$regex` | 正则匹配（不区分大小写） | `{"名称": {"$regex": "登录.*测试"}}` |
| `$like` | 模糊匹配（自动加 %） | `{"名称": {"$like": "登录"}}` |

### 元素

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `$exists` | 字段是否存在 | `{"备注": {"$exists": true}}` |
| `$size` | 数组长度 | `{"标签": {"$size": 3}}` |

### 逻辑

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `$and` | 逻辑与 | `{"$and": [条件1, 条件2]}` |
| `$or` | 逻辑或 | `{"$or": [条件1, 条件2]}` |
| `$not` | 逻辑非 | `{"$not": {条件}}` |
| `$nor` | 全部不满足 | `{"$nor": [条件1, 条件2]}` |

## 连表查询（$lookup）

通过 `lookup` 字段实现跨集合关联查询。系统会根据 `page_configs` 中的字段定义自动识别关联类型。

### 语法

```json
{
  "from": "目标集合标识",
  "localField": "本集合中的关联字段名（或中文标签）",
  "as": "结果中的列名"
}
```

### 支持的关联类型

| 关联类型 | 说明 | 返回格式 |
|----------|------|----------|
| relation（M:N） | 多对多关联，通过 `data_relations` 表 | 数组 |
| reference（父子） | 本地字段存储父记录 ID | 单个对象 |
| quoteSelect（引用选择） | 本地字段存储 ID 数组 | 数组 |

## 查询案例

### 当前系统集合关系图

```
巡检用例 (inspection-case)
  ├── [relation M:N] 模板id集合 ──> 巡检模板 (c38df47f)
  └── [relation M:N] 风险模板ID集合 ──> 风险模板 (f6b8f361)

PPT配置 (288c3ebc)
  ├── [reference 父子] 用例ID ──> 巡检用例 (inspection-case)
  └── [quoteSelect] 模板ID ──> 巡检模板 (c38df47f)

irisk数据 (a23a385c)
  └── [reference 父子] 用例ID ──> 巡检用例 (inspection-case)
```

---

### 案例 1：基本查询 - 按用例ID模糊搜索

```json
{
  "collection": "inspection-case",
  "query": {
    "用例ID": { "$regex": "IC" }
  },
  "sort": { "用例ID": 1 },
  "limit": 100
}
```

查出用例ID包含 "IC" 的巡检用例，按用例ID升序。

---

### 案例 2：单表连表 - 巡检用例 + 关联的巡检模板

```json
{
  "collection": "inspection-case",
  "query": {
    "用例ID": { "$regex": "IC" }
  },
  "lookup": [
    {
      "from": "c38df47f",
      "localField": "模板id集合",
      "as": "巡检模板"
    }
  ],
  "sort": { "用例ID": 1 },
  "limit": 100
}
```

查出匹配的巡检用例，并将每条用例通过 M:N 关联的**巡检模板**展开到「巡检模板」列。

---

### 案例 3：多表连表 - 同时关联模板和风险模板

```json
{
  "collection": "inspection-case",
  "query": {},
  "lookup": [
    {
      "from": "c38df47f",
      "localField": "模板id集合",
      "as": "巡检模板"
    },
    {
      "from": "f6b8f361",
      "localField": "风险模板ID集合",
      "as": "风险模板"
    }
  ],
  "select": ["用例ID", "用例名称", "用例类型"],
  "limit": 50
}
```

查出全部巡检用例，同时展开两个关联集合。`select` 控制只返回 3 个业务字段 + 2 个 lookup 列。

---

### 案例 4：反查父记录 - PPT配置 → 所属巡检用例

```json
{
  "collection": "288c3ebc",
  "query": {
    "操作类型": { "$regex": "检查" }
  },
  "lookup": [
    {
      "from": "inspection-case",
      "localField": "用例ID",
      "as": "所属用例"
    }
  ],
  "limit": 100
}
```

查出操作类型包含"检查"的 PPT 配置记录，通过 reference 关联将父级巡检用例信息展开。

---

### 案例 5：引用选择连表 - PPT配置 → 引用的模板

```json
{
  "collection": "288c3ebc",
  "query": {},
  "lookup": [
    {
      "from": "inspection-case",
      "localField": "用例ID",
      "as": "所属用例"
    },
    {
      "from": "c38df47f",
      "localField": "模板ID",
      "as": "引用模板"
    }
  ],
  "limit": 100
}
```

PPT配置同时连查两个集合：通过 reference 获取父级用例，通过 quoteSelect 获取引用的模板列表。

---

### 案例 6：组合条件查询（$or + 模糊匹配）

```json
{
  "collection": "inspection-case",
  "query": {
    "$or": [
      { "用例类型": "功能巡检" },
      { "优先级": "高" }
    ],
    "用例名称": { "$like": "登录" }
  },
  "sort": { "用例ID": -1 },
  "limit": 200
}
```

查出 (类型=功能巡检 **或** 优先级=高) **且** 名称模糊匹配"登录" 的用例，按用例ID降序。

---

### 案例 7：字段存在性检查 + 不等于

```json
{
  "collection": "inspection-case",
  "query": {
    "用例描述": { "$exists": true },
    "用例类型": { "$ne": "废弃" }
  },
  "select": ["用例ID", "用例名称", "用例描述"],
  "limit": 200
}
```

查出有描述且类型不为"废弃"的用例，只返回 3 个字段。

---

### 案例 8：多值匹配（$in）

```json
{
  "collection": "inspection-case",
  "query": {
    "用例类型": {
      "$in": ["功能巡检", "安全巡检", "性能巡检"]
    }
  },
  "sort": { "用例类型": 1, "用例ID": 1 },
  "limit": 500
}
```

查出类型为三种之一的用例，按类型、用例ID双重排序。

## 注意事项

1. **字段名支持中文**：查询条件、排序、投影中的字段名可以使用界面上显示的中文标签（如 `用例ID`），系统自动转换为内部字段名（如 `caseid`）
2. **数值比较**：使用 `$gt`、`$lt` 等操作符时，传入数字类型（如 `18`），系统会自动做数值转换
3. **正则语法**：`$regex` 使用 PostgreSQL 正则语法，默认不区分大小写
4. **结果上限**：单次查询最多返回 2000 条记录，建议配合分页使用
5. **连表性能**：lookup 会产生额外的数据库查询，大数据量下建议先用 `query` 缩小范围再连表
