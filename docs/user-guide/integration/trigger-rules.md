# 触发规则（数据联动）

触发规则在数据写入时自动执行**联动动作**，用于在记录变化时同步更新其它字段或记录。与 Webhook（外发 HTTP）不同，触发规则作用于**系统内部数据**。入口为 **设置中心 → 集成对接（`/admin/integration`）→ 触发规则**，需要管理员权限。

## 规则构成

- **触发集合 / 事件**：在某集合的 create / update / delete 时触发。
- **触发条件**（可选）：仅当记录满足条件时才执行。
- **联动动作**：满足条件后要执行的操作（如更新本记录或关联记录的字段）。

## 创建规则

1. 进入 **集成对接 → 触发规则**，点击 **新建规则**。
2. 选择 **触发集合** 与 **事件**。
3. 设置 **条件**（使用记录的原始字段名）。
4. 定义 **动作**：要写入的目标字段与取值。
5. 保存后即对后续写入生效。

## 与其它能力的区别

| 能力 | 作用对象 | 典型场景 |
|------|----------|----------|
| 触发规则 | 系统内部数据联动 | 改 A 字段时自动更新 B 字段 |
| [Webhook](./webhooks.md) | 外发到第三方 HTTP | 通知外部系统 |
| [工作流](../data/workflow.md) | 跨页阶段流转 | 状态推进生成下游记录 |

## 提示

- 条件与动作均使用**原始字段名**。
- 避免规则互相触发形成循环联动。
