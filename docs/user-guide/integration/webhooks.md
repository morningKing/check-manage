# Webhook 规则

Webhook 让系统在数据发生变化时，自动向**第三方 HTTP 端点**推送事件（带 HMAC-SHA256 签名）。入口为 **设置中心 → 集成对接（`/admin/integration`）→ Webhook**，需要管理员权限。

## 触发事件

| 事件 | 触发时机 |
|------|----------|
| `create` | 集合中新增记录 |
| `update` | 记录被修改 |
| `delete` | 记录被删除 |
| `merge` | 分支合并 |

每条规则绑定一个或多个 **集合（collection）** 与上述事件。

## 创建规则

1. 进入 **集成对接 → Webhook**，点击 **新建规则**。
2. 填写：
   - **目标 URL**：接收事件的第三方端点；
   - **监听集合 / 事件**：选择触发的集合与事件类型；
   - **密钥 Secret**：用于生成签名；
   - **触发条件**（可选）：JSON 过滤器，仅当记录满足条件才触发，例如 `{"status": "completed"}`；
   - **超时 / 重试**：请求超时与失败重试次数。
3. 保存后规则即生效。

## 签名校验

每次请求头携带 `X-Webhook-Signature`，其值为 **payload 的 HMAC-SHA256**（使用规则的 Secret）。接收端应以相同算法重新计算并比对，确认请求来自本系统、未被篡改。

## 测试与日志

- **测试执行**：在规则上点击「测试」，系统发送一条样例 payload 验证连通性。
- **执行日志**：`webhook_logs` 记录每次触发的请求 / 响应 / 重试情况，可在规则详情查看，用于排查失败。

## 常见问题

- **收不到回调**：检查目标 URL 可达、防火墙、以及触发条件是否过滤掉了事件。
- **签名校验失败**：确认两端 Secret 一致、且对**原始 payload 字节**计算签名。
